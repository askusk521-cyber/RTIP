# 假设（中文版）

## 已确认项

- 本次迁移不移植 CP2K。
- CP2K 的输入/输出语义需保留文档，以便将来由其他 PES 提供者替换。
- 核心数值代码优先使用 `jax.numpy` 与 JAX 原语。
- 默认启用 JAX x64，以尽可能匹配 Rust 的 `f64`。
- 随机行为使用显式 JAX PRNG keys。
- 工作流与 IO 可保持普通 Python。

## 工作假设

- Python 包名为 `rtip_jax`，发行名 `rtip-jax`。
- IDWM 为优先迁移对象，因为它为纯数值模块且不依赖特定求解器符号。
- 初期不需要 MPI；如需分布式，将在工作流层隔离处理。
- CP2K 相容的单位保留为内部基线，除非后阶段另有说明。
- 本地环境可能不包含 JAX/pytest，依赖的检查推迟到远端验证环境。

- 是否保留 2-molecule / 3-molecule / 4-molecule 的独立入口？
- 哪些 Rust 夹具或示例应作为数值基线？
