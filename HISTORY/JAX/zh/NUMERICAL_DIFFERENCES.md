# 数值差异（中文版）

记录 Rust 与 JAX 实现间已知或预期的数值差异。

## 当前状态

- Stage 3 添加了常数、配置、System IO 与随机旋转。
- Stage 4.1 添加了 IDWM，保留了上三角约定和零距离归一化奇异性。
- Stage 4.2 添加了 RTIP，保留四元数矩阵与转换公式，但特征向量可能受后端影响。
- Stage 4.3 添加了优化，保留 Rust 的一维搜索策略，但在零总力方向上显式拒绝以避免 NaN。

## 预期差异区域

- JAX 与 OpenBLAS/LAPACK 可能在 RTIP 特征向量符号上不同。
- RTIP 的简并或近简并情形可能导致选向向量差异。
- JAX x64 可减小但不能完全消除与 Rust `f64` 的差异。
- IDWM 与 RTIP 的力公式在零距离处有除法操作，零距离需专项测试。
- 随机旋转与初始扰动不会复制 Rust 的随机流；JAX 端使用显式 key 可复现。

## 容忍策略草案

- 纯代数辅助：严格或接近机器精度。
- RTIP/IDWM 能量与力：初期测试容差 `rtol=1e-10`, `atol=1e-12`。
- 工作流轨迹比较以不逐位相同为前提，比较不变量与标量序列容差。
