# 测试策略（中文版）

## Stage 2 测试

- 包导入时配置 JAX x64。
- CP2K 边界记录原始输入/输出/单位契约。

## 本地与远端执行

当前本地环境可能不含 JAX 或 pytest，依赖相关检查推迟到远端验证服务器。局部验证限于不依赖第三方包的语法与元数据检查。

## Stage 3 测试要点

- 常数与单位换算匹配 Rust。
- 元素解析与质量匹配 Rust 行为。
- `Para` 默认值匹配 Rust 的 `Para::new()`。
- XYZ 读写在 Angstrom <-> Bohr 的转换可逆。
- 随机旋转在固定 JAX key 下可复现且正交。

## Stage 4 测试要点

- IDWM 的加权矩阵与距离校验、力方向数值差分、片段力散布等。
- RTIP 的距离不变性、变换重构、力方向数值差分、片段力散布等。
- 优化的一维搜索与力方向位移、零力拒绝。
- 偏置 PES dataclass 验证与 `SumPES` 求和。
- 合成布局的 2/3/4 分子偏移与确定性测试。

## Stage 5 测试要点

- JSON 参数加载与未知键校验。
- 输出辅助路径对 legacy filename 的支持。
- CLI 烟雾测试（默认配置与 CP2K 边界检查）。
- 路径采样与 MD 运行器的确定性 mock PES 跑通测试。

## 所需基线

- 小型 XYZ 夹具用于 1、2、3 分子情形。
- 一旦达成约定，使用 Rust 生成的标量快照作为 IDWM 与 RTIP 的基线。
