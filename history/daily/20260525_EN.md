## Design

### Core Idea: Replace Hard-Coding with Automatic Detection

Do not specify atom indices and reorder mappings separately for each reaction. Instead, use element types and distance-based geometric heuristics for automatic detection.

### Small-Molecule Type Identification

Automatically identify the type from the element composition of fragment 2 (`2.xyz`):

| Atom count | Element counts | Type | B-bonding element |
|------------|----------------|------|-------------------|
| 3 | C:1 O:2 | CO2 | O |
| 3 | C:1 S:2 | CS2 | S |
| 4 | C:1 O:1 H:2 | H2CO | O |
| 6 | C:2 N:1 H:3 | MeCN | N |
| 11 | C:3 N:1 H:7 | MeCH=NMe | N |

### Automatic Detection of Reactive Atoms

**In the IS system (fragment 1 + fragment 2 concatenated)**:

- **B atom**: the unique boron atom, found directly through `atom_type == Element.B`
- **N2 pair**: the two N atoms in fragment 1 (`1-R-N2` complex) closest to B. All fragment-1 systems have 4 N atoms (2 NHC ring N atoms + 2 N2 atoms), and the two N2 atoms are directly bonded to B, so they are the nearest to B
- **Small-molecule C**: the carbon atom in fragment 2. For CO2/CS2/H2CO there is only one C; for MeCN (2 C) / MeCH=NMe (3 C), choose the C farthest from B (the electrophilic carbon is farther from the reaction center)
- **Small-molecule X (the atom bonding to B)**: atoms in fragment 2 matching the small-molecule type (O/S/N), sorted by distance to B

**In the reference structure (`product.xyz` / `ts.xyz`)**:

- **B atom**: same as above
- **N2 pair**: identified by the shortest N-N distance (N2 triple bond ~1.2 A, far shorter than any other N-N distance). This also works for N-containing small molecules (MeCN, MeCH=NMe), because the N2 triple bond is always the shortest N-N distance in the system
- **Small-molecule C**: the C atom closest to the N2 pair among all C atoms (in product/TS, the terminal N of N2 has formed or is forming an N-C bond to the small-molecule carbon)
- **Small-molecule X**: the O/S/N atom nearest B after excluding N2 atoms

### Core RMSD Calculation

Do not perform full-atom reordering. Instead, extract 5-6 reactive-core atoms (B + N2 x2 + small-molecule C + small-molecule X x1~2), detect these atoms independently in simulation frames and reference structures, then compute Kabsch-aligned RMSD.

### RC Restraint Construction

From the target reference structure (`product.xyz`):
- N-C restraint: nearest distance between terminal N2 nitrogen and small-molecule C
- B-X restraint: nearest distance between B and small-molecule X atoms

The two harmonic distance restraints use the same k value.

## Issues Found and Fixed During Tuning

### Issue 1: No Local JAX Environment

`rcmd_lib.py` initially used `import jax.numpy as jnp` for coordinate concatenation, but local Python did not have JAX installed. Fix: use `numpy` for concatenation (`np.concatenate`); the `System` class accepts numpy arrays as the `coord` argument. JAX-related imports remain only in `run_rcmd.py` (runner), which runs only in the n5 deepmd environment.

### Issue 2: Incorrect Element Signature for MeCH=NMe Fragment

The initial signature was written as `C:3, N:2, H:7` (12 atoms total), but actual MeCH=NMe (CH3-CH=N-CH3) is C3H7N (`C:3, N:1, H:7` = 11 atoms). It is correctly identified after the fix.

### Issue 3: Reference N2 Detection Conflict for N-Containing Small Molecules

MeCN and MeCH=NMe systems have 5 N atoms total (4 from the 1-tBu fragment + 1 from the small molecule). The original heuristic "two N atoms nearest B = N2 pair" failed in the MeCN system: the small-molecule N also bonds to B in product and was inserted between the two N2 N atoms when sorted.

Fix: identify the N2 pair by shortest N-N distance. The N2 triple bond length is ~1.2 A, much shorter than any other N-N distance (NHC ring N -> N2 > 2.5 A, MeCN N -> N2 > 2.5 A). This is correct for all systems, including pure CO2 systems with 4 N atoms.

At the same time, explicitly exclude the identified N2 atoms when detecting small-molecule X, preventing an N2 N atom from being misclassified as the small-molecule reactive N.

### Issue 4: Ordering Uncertainty in IS `small_x_indices`

In the IS geometry, the two O atoms of CO2 are at nearly equal distances from B (~5 A). The stability of `sorted()` and tiny coordinate differences can make the order switch between `(34,35)` and `(35,34)`. This does not affect actual calculations because `_min_b_x()` takes the minimum distance over all `small_x_indices`, and the RC restraint uses the specific atom pair determined by `_closest_pair` in the reference structure.

## Discussion and Decision Record

1. **Do not use full-atom reorder mappings**: the original runner's `REFERENCE_REORDER_1TBU_CO2` is a manually derived 36-atom permutation. Repeating this for every new reaction is costly and error-prone. The new approach independently detects reactive-core atoms in the reference structure and simulation structure; it only needs element matching for 5-6 atoms and does not require knowing the complete permutation.

2. **Keep existing files unchanged**: `run_bias_exploration_1tbu_co2.py` and `analyze_bias_exploration_1tbu_co2.py` are kept as validated reference anchors. If the generalized version has problems, the original version can still independently validate 1-tBu__CO2.

3. **Use k=0.006 uniformly for all reactions**: this force constant has units of Hartree/Bohr^2 and is only related to distance restraints, not the total atom count. In theory it should transfer across systems. If some systems do not behave well, tune them separately afterward.

4. **Do not use TS target**: based on yesterday's conclusion, product target has B-X and N-C distance directions that are consistent, allowing RC restraint to constrain both effectively. The two TS-target distances point in inconsistent directions, making trajectories unstable.
