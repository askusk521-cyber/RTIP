### 失败模式分析

k=0.006 仅在 1-tBu__CO2 上有效，对其余 10 个反应均不足以驱动反应完成。存在两种不同的失败模式：

**模式 A：高温失控（小体系 CO2 反应）**

1-Me__CO2、1-H__CO2、1-CN__CO2 的初始 rti_dist 很大（8.3~9.6 A），体系在 restraint 拉动力下向目标快速移动，但 DeePMD PES 产生排斥力，动能累积导致温度急剧升高（峰值 1237~1302 K）。thermostat（target 300 K, thermo_lambda ~0.999）来不及移除热量，体系以高温振荡而非收敛到 product 盆地。

这些体系的共同特征：取代基小（H, Me, CN），体系总原子数少（24~27），缺乏足够的内部自由度来耗散 restraint 注入的能量。

**模式 B：冷停滞（大取代基反应）**

1-PMe2__CO2、1-Ph__CO2、1-SiMe3__CO2 的初始 rti_dist 较小（1.3~4.0 A），意味着 IS 几何中片段位置已经较接近 product。但 restraint 力太小（k=0.006），体系在接近起始位置的区域徘徊，温度偏低（195~341 K），没有足够动能跨越 PES 上的重排势垒。最优成键帧出现在 step 110~124（轨迹早期），之后体系漂移远离。

**模式 B'：中等升温（1-tBu 系列非 CO2 反应）**

1-tBu__CS2、1-tBu__H2CO、1-tBu__MeCN、1-tBu__MeCH=NMe 的初始 rti_dist 中等（3.8~5.4 A），restraint 有一定拉力，温度适度升高（335~509 K），但不足以到达 product 区域。最终 B-X 和 N-C 距离均远超参考 product 值。

### 为什么只有 1-tBu__CO2 成功？

1-tBu__CO2 的组合优势：
1. **tBu 取代基的空间位阻**：提供适度的空间引导，帮助 CO2 定向接近 B 中心
2. **CO2 的线性几何**：两个 O 原子对称分布，增加了与 B 成键的概率
3. **合适的 IS 初始距离**：rti_dist = 7.97 A，restraint 有足够的"拉动力臂"
4. **tBu 取代基的大自由度**：36 个原子提供足够的内部振动模式耗散多余动能

其他 1-tBu 系列反应失败说明**小分子的电子结构和几何对反应路径有决定性影响**：
- CS2：S 原子半径大，B-S 键更长，k=0.006 不足以克服 S 的空间排斥
- H2CO：只有一个 O，且 H2CO 为平面结构，空间取向要求更苛刻
- MeCN/MeCH=NMe：B-N 键形成涉及不同的电子结构，且含 N 小分子在 IS 中可能与 N2 配位竞争

### promising 标志的可靠性

仅 3/11 反应标记为 `promising=True`（1-tBu__CO2, 1-Me__CO2, 1-CN__CO2），但 1-Me__CO2 和 1-CN__CO2 的最终 B-X/N-C 距离均远未达到成键阈值。`promising_bond_threshold_met` 的判断依据需要复核——可能是阈值设定过于宽松，或是使用了轨迹中某帧的最优值而非最终帧。

## 下一步建议

1. **增大 k 值**：对失败体系尝试 k=0.01, 0.02, 0.05，寻找每个反应的最小有效 k
2. **增加步数**：对 rti_dist 持续下降但未收敛的反应（如 1-tBu__H2CO），尝试 2000~5000 步
3. **渐进式 restraint**：从较弱的 k 开始，逐步收紧到目标距离，避免初期高温失控
4. **对 1-tBu 系列单独调参**：固定 fragment 1 相同，系统性地研究小分子类型对最优 k 的影响
5. **复核 promising 阈值**：调整 `promising_bond_threshold_met` 的判断逻辑，确保其与实际产物接近程度一致

## 三阶段综合结论

**以下结论建立在"rc-md product-target, 300K thermostat, DPA-3.2-5M 模型"这一条件基础上：**

### 1. 只有 1-tBu__CO2 在 k=0.006 成功——不是 k 不够大

上午的分析将失败归因于 k=0.006 太小。但 Phase 1 的 k 扫描表明，**k 从 0.006 增大到 0.05（8.3 倍）对 10 个失败反应中的任何一个都没有实现产物态**。这说明问题不是 restraint 强度，而是：
- 小体系：增大 k 导致温度失控，体系脱离物理合理的 PES 区域
- 大体系：增大 k 仍不足以克服空间位阻/电子结构差异带来的势垒

### 2. B-X 成键比 N-C 成键容易得多

在所有反应中，B-X 距离（B-O/B-S/B-N）在足够 k 下可降至参考值附近，但 N-C 距离始终难以收敛。这可能反映了一个机理事实：**B-X 键形成是反应的第一步（亲核进攻），而 N-C 键形成（协同的环加成或分步的后续重排）涉及更高的能垒**。当前 1D restraint（B-X + N-C 距离之和）无法解耦这两个过程。

### 3. 温度是核心限制因素

几乎所有失败案例都涉及温度问题：
- **温度过高**（模式 A）：体系获得过多动能，无法被 thermostat 及时移除
- **温度过低**（模式 B）：restraint 注入的能量不足以激活重排

渐进式 restraint（Phase 3）是缓解此问题的有效策略，但仍不足以完全解决。

### 4. 取代基效应决定反应可行性

以 1-tBu 为参照的对比：

| 小分子 | k=0.006 最终 score | 最佳 k | 最佳 best_score | 可行性评价 |
|--------|------------------:|-------:|---------------:|-----------|
| CO2 | 2.684 | 0.006 | 2.684 | **唯一成功** |
| H2CO | 6.996 | 0.05 | 3.505 | 接近，最有希望的非 CO2 |
| CS2 | 7.629 | 0.05 | 5.642 | 中等，2000步有改善 |
| MeCN | 9.075 | 0.05 | 6.537 | 困难 |
| MeCH=NMe | 7.256 | 0.02 | 3.569 | 中等偏难 |

CO2 的线性结构和两个对称 O 原子提供了独特的优势——无论从哪个角度接近 B 中心，至少有一个 O 可以进行亲核进攻。

### 5. 下一步建议（按优先级）

1. **对 1-H__CO2 和 1-Me__CO2 扩展渐进式 restraint**：当前 3 阶段（0.006→0.01→0.02）已取得温度控制的好效果，可尝试 4~5 阶段延伸到 k=0.05，总步数 2000~3000
2. **对 1-tBu__H2CO 尝试 k=0.05 + 渐进式 + 2000 步**：k=0.05 时 best 帧已接近成键（score=3.505），但单阶段导致 T=1287K，渐进式可能解决温度问题
3. **对 CO2 系列尝试 2D restraint**：考虑分离 B-X 和 N-C 的 restraint 时间表——先驱动 B-X 成键（Stage 1），再驱动 N-C 成键（Stage 2）
4. **放弃或根本改变策略的反应**：1-Ph__CO2, 1-PMe2__CO2（k=0.05 几乎无响应）；1-tBu__MeCN, 1-tBu__MeCH=NMe（B-N 键形成对当前方法极不敏感）
5. **复核 promising 标志的阈值**：1-CN__CO2 的三个 k 值均标记为 promising=True，但所有结果的 N-C 距离均远超参考——阈值需要重新校准
6. **对 1-tBu__CS2 k=0.01 尝试 5000 步**：2000 步已有 N-C 改善趋势（4.527→3.883），且最佳帧在 step 1252，说明更长轨迹可能有利

## "1D restraint" 的代码级解释

### 为什么叫 "1D"

RC-MD 的 restraint 由两个独立的距离组成（B-X 和 N-C），但**共享同一个力常数 k**。可调的只有一个旋钮，所以称为"1D"。

代码定义在 [rcmd_lib.py:352-369](research/ic5c02384/scripts/rcmd_lib.py#L352-L369)：

```python
def reaction_coordinate_restraints(target, config, k):
    # Restraint 1: N-C 距离
    DistanceRestraint(n_atom, c_atom, n_c_dist, k, "N-C")
    # Restraint 2: B-X 距离  
    DistanceRestraint(b_atom, x_atom, b_x_dist, k, "B-X")
    # ← 两个 restraint 用的是同一个 k
```

势函数（[bias.py:121-138](rtipmd/jax/src/rtip_jax/pes/bias.py#L121-L138)）是两个独立谐振子之和：

```
E_bias = ½k(d_BX - target_BX)² + ½k(d_NC - target_NC)²
```

输出中的 `rti_dist`（distance_column）是两个偏差的 RMS：

```
rti_dist = sqrt((d_BX - target_BX)² + (d_NC - target_NC)²)
```

### 1D 的局限性

增大 k → B-X 和 N-C 的拉动力**同步增强**。但 B-X 成键（亲核进攻）和 N-C 成键（后续重排）是动力学上两步不同的过程。1D 设计无法分别控制——为了让 N-C 靠近而继续增大 k，B-X 方向已经被过度拉紧，多余能量以热的形式释放，导致温度失控。

这引出了下一步建议中的 "2D restraint"：给 B-X 和 N-C 各自独立的 k 和时间表，例如先驱动 B-X 成键（k_BX 大，k_NC 小），再驱动 N-C 成键。

## JAX 版本的 RTIPMD 双轨制：RTIP 高斯 bias vs RC-MD 谐振子 restraint

经过对 JAX 和 Rust 源码的对比阅读，发现 JAX 的 `bias.py` 实际上包含了两套**完全独立**的 bias 机制，通过 `md.py:254-263` 的 dispatch 共用同一个 MD 引擎（leapfrog + Berendsen thermostat）：

```python
# md.py:254-263 _md_bias()
if isinstance(config, RepulsivePot):         → RTIP 高斯排斥
if isinstance(config, AttractivePot):        → RTIP 高斯吸引
if isinstance(config, SynthesisPot):         → RTIP 高斯合成
if isinstance(config, ReactionCoordinatePot): → RC-MD 谐振子 ← 仅 JAX 有
```

### 机制 1：RTIP 高斯 bias（JAX 移植自 Rust，完全对应）

**Config**: `RepulsivePot` / `AttractivePot` / `SynthesisPot`（[bias.py:18-61](rtipmd/jax/src/rtip_jax/pes/bias.py#L18-L61)）
**PES**: `Rtip0PES`（[core/rtip.py:196-260](rtipmd/jax/src/rtip_jax/core/rtip.py#L196-L260)）
**反应坐标**: RTI 距离——两个构型做最优旋转/平移对齐后的 RMSD（四元数法，对 4×4 矩阵做特征值分解取最小特征值）

势函数（[core/rtip.py:159-167](rtipmd/jax/src/rtip_jax/core/rtip.py#L159-L167)）：

```
pot = Σ (w_i / Σw) * a * exp(-r_i² / 2σ²)
w_i = 1 / r_i⁷    # 权重函数，近的构型主导
```

参数调度（[md.py:206](rtipmd/jax/src/rtip_jax/workflows/md.py#L206)）：
```python
a_min = config.para.a0 * float(step)    # 排斥: 正振幅随步数线性增长
a_min = -config.para.a0 * float(step)   # 吸引: 负振幅（吸引阱加深）
```

核心特征：
- 1 个反应坐标（RTI 距离），1 个可调参数（a 振幅，随时间增长）
- 势函数是高斯：距离很远时力反而减小（有最优力区间），不会无限拉
- σ 每步动态更新 = 当前构型到参考构型的 RTI 距离
- 三种模式：排斥（从极小值推出）、吸引（拉向目标）、合成（多分子组装）
- 支持 `atom_add_pot`：只对部分原子施加 bias（[system.py](rtipmd/jax/src/rtip_jax/system.py)）

### 机制 2：RC-MD 谐振子 restraint（仅 JAX 存在，Rust 中无对应）

**Config**: `ReactionCoordinatePot`（[bias.py:151-170](rtipmd/jax/src/rtip_jax/pes/bias.py#L151-L170)）
**PES**: `ReactionCoordinatePES`（[bias.py:105-148](rtipmd/jax/src/rtip_jax/pes/bias.py#L105-L148)）
**反应坐标**: 两个具体的原子间距离（B-X 和 N-C）

关键：`_reaction_coordinate_md_bias`（[md.py:248-251](rtipmd/jax/src/rtip_jax/workflows/md.py#L248-L251)）**忽略 step 参数**，k 始终不变。

核心特征：
- 2 个距离坐标，1 个共享 k，固定 target
- 势函数是谐振子：距离越远力越大（→ 必然温度失控）
- 仅吸引模式（target = 参考产物构型的距离）
- 自动检测原子索引（`detect_reaction_config()` in rcmd_lib.py）

### 并排对比

| | RTIP 高斯 (JAX & Rust) | RC-MD 谐振子 (仅 JAX) |
|---|---|---|
| **config 类型** | `RepulsivePot` / `AttractivePot` / `SynthesisPot` | `ReactionCoordinatePot` |
| **PES 类** | `Rtip0PES` (core/rtip.py:197) | `ReactionCoordinatePES` (bias.py:106) |
| **反应坐标** | 1 个 RTI 距离（全局构型 RMSD） | 2 个原子间距离（B-X, N-C） |
| **势函数** | 高斯 `a·exp(-r²/2σ²)` | 谐振子 `½k(r-r₀)²` |
| **参数调度** | a = a0 × step（线性增长） | k 固定不变 |
| **目标定义** | 参考构型的坐标（全局） | 两个距离的标量 target 值 |
| **dof 数量** | 1 个 RTI 距离 | 2 个独立距离 |
| **可调参数** | a0, σ, pot_climb, pot_drop | k（唯一参数） |
| **Rust 对应** | rtip.rs + idwm.rs | **不存在** |
| **输出列名** | rti_dist（RTI 距离） | rti_dist（reused，实际是 distance_error RMS） |

### 为什么叫 rti_dist

MD 输出格式是按 RTIP 高斯 bias 设计的（[md.py:170](rtipmd/jax/src/rtip_jax/workflows/md.py#L170)），列名硬编码为 `rti_dist`。RC-MD 复用时，`_reaction_coordinate_md_bias` 把 `bias_pes.distance_error(system)`（两个谐振子偏差的 RMS）塞进了同一个列位置。所以 RC-MD 输出的 "rti_dist" 实质上是 `sqrt((d_BX-target_BX)² + (d_NC-target_NC)²)`。

### 对 "1D restraint" 命名的修正

RC-MD 的 "1D" 应更准确地描述为 **"single-k, 2-distance harmonic restraint"**。Rust/JAX 的 RTIP 高斯 bias 才是真正的 **"1D"（1 个 RTI 距离，1 个增长参数 a）**。但 RTIP 高斯的不足是它不知道"哪根键形成了"——整个反应进程被压缩为一个标量，无法像 RC-MD 那样分别观察 B-X 和 N-C 的成键进度。

## RTIP 高斯 bias 为何能驱动小分子聚合反应

### 自适应 sigma 是关键设计

之前的分析中有一个重要的纠正：RTIP 高斯 bias **不是**固定宽度的。sigma 每步被设为当前体系到参考构型的 RTI 距离本身：

```python
# md.py:218 (_attractive_md_bias) / md.py:235 (_synthesis_md_bias)
sigma_min = rti_dist(_coords(config.final_state, indices), _coords(system, indices))
```

这意味着在高斯函数 `a * exp(-r²/2σ²)` 中，r ≈ σ（第一个 RTI 距离等于 sigma），所以：

```
exp(-r²/2σ²) = exp(-σ²/2σ²) = exp(-1/2) ≈ 0.607  恒定！
```

**指数项永不衰减。** 无论碎片离参考点多远，高斯的指数部分始终是常数 0.607。

力的主体项（忽略权重再分配）：
```
F ∝ a / σ² * exp(-1/2) = a0*step / r² * 0.607
```

- 力按 **~1/r²** 衰减（温和的平方反比），不是指数衰减
- a 随步数线性增长 → 力持续补偿
- 体系无法"跑赢" bias，因为 sigma 自适应跟踪距离（棘轮效应）
- 对固定 sigma 的高斯，远处力→0；对 RTIP 自适应 sigma，远处力→`a0*step/r²`

### 固定 sigma vs 自适应 sigma 对比

| | 固定 σ 高斯 | RTIP 自适应 σ (sigma=r) |
|---|---|---|
| 指数项 | `exp(-r²/2σ²)` → 0 (远处) | `exp(-1/2)` = 0.607 恒定 |
| 力衰减 | 指数级崩塌 | ~1/r² (温和) |
| 远处行为 | 力消失，拉不动 | 力虽弱但存在，持续向外推 |
| 配合 step 增长 a | — | 力随时间线性补偿 |

### 为什么 Repulsive 也能用于聚合反应

Rust 原版 MD 层只有 `impl RtipNVTMD for RepulsivePot`（[md.rs:18](rtipmd/rust/src/pes_exploration/md.rs#L18)），但框架设计哲学是"逃离极小点"（[lib_2mol.rs:1-6](rtipmd/rust/src/lib_2mol.rs#L1-L6)）：

> RTIP is a biased potential to drive the molecule escaping from local minimum

对聚合反应，**单体的孤立态就是一个 local minimum**。Repulsive bias 推离单体 → 体系必然探索非单体构型 → 包含二聚/寡聚/聚合态。逃离单体 basin 本身就是聚合。

在 MD 中有热涨落辅助：Repulsive bias 提供持续向外的"搅拌力"，热运动提供正交方向的探索，形成**有偏随机行走**（类似 metadynamics 但 bias 形式不同）。自适应 sigma 保证力永不消失，step 增长保证力持续增强，形成棘轮效应——一旦体系偏离单体，bias 就把它推得更远。

`atom_add_pot`（[md.rs:38-65](rtipmd/rust/src/pes_exploration/md.rs#L38-L65)）允许只对反应位点原子施加 bias，让 RTI 距离更直接地对应反应坐标，忽略旁观原子的噪声。

### Pathway 层 vs MD 层

Rust trait 体系（[traits.rs](rtipmd/rust/src/pes_exploration/traits.rs)）将两者定义为独立的接口：

```rust
pub trait RtipPathSampling { ... }  // Pathway 接口
pub trait RtipNVTMD { ... }         // MD 接口
```

| | Pathway 层 | MD 层 |
|---|---|---|
| **温度** | 0 K（temp=nan, kin=nan） | 有限温度，Berendsen 恒温器 |
| **运动方式** | 沿合力方向 1D 线搜索（黄金分割），直接位移 | 蛙跳积分推进坐标+速度，Berendsen thermostat 控温 |
| **速度** | 无速度概念 | 有速度，服从麦克斯韦分布 |
| **停判** | 状态机：pot_drop 关 bias → f_epsilon 收敛停 | 固定步数，跑满 max_step |
| **目标** | 找到最小能量路径（类似 IRC/GSM） | 模拟有限温度下的真实动力学轨迹 |
| **随机性** | 仅初始扰动，后续确定性线搜索 | 全程热涨落，随机行走 |
| **Rust 实现** | Repulsive/Attractive/Synthesis 三种都有 | **只有 Repulsive** |

原版 Rust 设计的分工：
1. **Pathway 层**：用 AttractivePot 或 SynthesisPot 找到聚合路径（碎片→产物，3 种 bias 都有）
2. **MD 层**：以路径起点为参考做有限温度采样/弛豫，Repulsive bias 防止重陷反应物 basin

MD 层不需要 Attractive/Synthesis，因为"往哪走"已经在 Pathway 层解决了。MD 只负责"在有限温度下走得怎么样"。

### 对 ic5c02384 而言为什么必须扩展 MD 层

对于 N2 + 小分子偶联反应：
- 碎片初始距离远，Pathway 的 0 K 线搜索（纯力驱动、无热辅助）找不到合理反应路径
- MD 必须自己"驱动"反应——不仅要采样，还要主动把碎片拉到一起
- Repulsive 只能推离单体，无法指定"哪两个原子要成键"
- → 必须在 MD 层加入 Attractive/Synthesis/RC-MD

这就是 week1conclusion.md 所说的："Attractive / synthesis MD 以及 ReactionCoordinate 是本次研究中发现 repulsive 走不通结合反应后，在 JAX 层现场扩展的。"

### RC-MD vs RTIP 高斯：适用场景

| | RTIP 高斯 (Attractive/Synthesis) | RC-MD 谐振子 |
|---|---|---|
| 力随距离变化 | ~1/r² 衰减（远处弱但存在） | ~r 线性增长（越远越强） |
| 适用场景 | 一般小分子对接/聚合（无精确键长目标） | 特定成键距离已知的结合反应 |
| 收敛速度 | 较慢，需要较多步数 | 较快，但 k 太大会温度失控 |
| 可解释性 | 需调 a0，物理含义间接 | k 即弹簧刚度，物理含义直接 |
| 通用性 | 高（不依赖特定原子对知识） | 低（需要事先知道哪对原子成键和 target 距离）

## 2026-05-26 晚间：JAX RTIP 高斯 bias 运行 1-tBu__CO2 方案设计

### 背景

此前所有 RC-MD 实验（Phase 1-3）使用的都是谐振子 restraint（`ReactionCoordinatePot`，`E = ½k(r-r₀)²`），这是 JAX 侧为 ic5c02384 论文专门添加的机制，Rust 原版中不存在。用户要求改用与 Rust 原版一致的 RTIP 高斯 bias 形式来跑 1-tBu__CO2 反应。

### RTIP 高斯 bias 的数学形式（Rust 原版）

```
E_bias = Σ_i (w_i / Σw) * a * exp(-r_i² / 2σ²)

其中：
  w_i = 1 / r_i⁷           # 权重函数，近构型主导
  a = a0 * step             # 振幅随步数线性增长（吸引时为负）
  σ = rti_dist(ref, cur)   # 自适应 sigma = 当前构型到参考的 RTI 距离
```

关键特征：
- **自适应 sigma**：`σ = rti_dist(ref, cur)`，使得指数项 `exp(-r²/2σ²) ≈ exp(-1/2) ≈ 0.607` 恒定
- **力衰减 ~1/r²**：温和的平方反比，不是指数衰减
- **棘轮效应**：sigma 自适应跟踪距离，体系无法"跑赢" bias
- **1 个反应坐标**（RTI 距离）+ 1 个可调参数（a0），真正"1D"

### 与 RC-MD 谐振子的对比

| | RTIP 高斯 (Rust 原版) | RC-MD 谐振子 (此前实验) |
|---|---|---|
| 势函数 | `a·exp(-r²/2σ²)` | `½k(r-r₀)²` |
| 反应坐标 | 1 个 RTI 距离（全局 RMSD） | 2 个原子间距离（B-X, N-C） |
| 可调参数 | a0（振幅增长率） | k（力常数，固定） |
| 力-距离关系 | ~1/r² 衰减（远处弱但持续） | ~r 线性增长（越远越强→温度失控） |
| sigma | 自适应（= RTI 距离） | N/A |
| 参数调度 | a 随 step 线性增长 | k 始终不变 |
| 远处行为 | 力虽弱但存在，持续拉/推 | 力发散，必然温度失控 |
| Rust 对应 | **存在**（rtip.rs + md.rs） | **不存在**（仅 JAX） |

### 问题梳理

#### 问题 1：CLI 不支持 Attractive/Synthesis MD

`cli.py` 的 `deepmd-pathway` 和 `deepmd-md` 子命令硬编码了 `RepulsivePot`，无法通过命令行运行 Attractive MD。必须通过 Python API 编写脚本调用。

#### 问题 2：Rust 原版只有 Repulsive MD

Rust `md.rs` 只实现了 `impl RtipNVTMD for RepulsivePot`，Attractive MD 和 Synthesis MD 是 JAX 侧扩展的。但"与 Rust 形式一致"指的是 **bias 函数形式**（高斯 + 自适应 sigma），JAX 的 `AttractivePot` MD 完全满足（仅 `a = -a0 * step`，其余一致）。

#### 问题 3：bias 参考构型选择

- **Repulsive 模式**：`local_min` = IS 分离构型，推离反应物 basin → 探索结合态
- **Attractive 模式**：`final_state` = product.xyz，拉向已知产物
- 两种模式数学形式相同，物理意义相反。对已知产物的反应，Attractive 更直接。

#### 问题 4：`atom_add_pot` 的取舍

RTI 距离计算全局构型 RMSD。如果对全部 36 个原子计算，旁观原子（tBu 上 H）的热涨落会稀释反应位点信号。Rust 支持 `atom_add_pot` 只选反应位点原子。但首次运行建议全原子 RTI 建立基线。

#### 问题 5：初始速度策略

- Rust 原版：零初始速度，仅坐标随机扰动（`scale=0.1`）
- 现有研究脚本：定向初始速度指向 target（`_target_velocity()`）
- 与 Rust 一致应选零初始速度

#### 问题 6：参数 `a0` 的选择

- Rust 默认：`a0 = 0.002`
- 研究脚本：`a0 = 0.0005`
- 无先验知识，需扫描

#### 问题 7：SynthesisPot vs AttractivePot

- **AttractivePot**：`final_state` = product.xyz，RTI 距离包含产物成键信息
- **SynthesisPot**：动态计算分子几何中心聚合态，不含键长信息
- 对有已知产物的反应，AttractivePot 更合适

#### 问题 8：Pathway 与 MD 的组合

Rust 设计哲学：Pathway（0K 线搜索）→ MD（有限温度采样）。但 JAX 的 AttractivePathway 和 AttractiveMD 是独立的。首次运行建议直接 MD，后续可考虑先 Pathway 后 MD。

#### 问题 9：运行环境

必须在 n5 服务器运行（DeePMD-kit 3.1.1 + CUDA 12.9.1 + DPA-3.2-5M.pt），本地无法运行 DeePMD。

### 分析：RTIP 对 1-tBu__CO2 为何不如 RC-MD

**RTIP Attractive bias (a0=0.002, 1000步) 未能将 1-tBu__CO2 驱动到产物态。RC-MD 谐振子 restraint (k=0.006, 1000步) 仍是该反应唯一成功的方法。**

三个根因：

**1. RTI 距离是"钝"反应坐标**

RC-MD 直接 restraint 定义反应的两个特定原子间距离（B-O 和 N-C）。RTIP 使用所有原子的全局构象 RMSD——它知道 CO2 片段应该靠近 tBu 片段，但不知道具体哪些原子应该成键。系统可以通过将 CO2 大致移近 tBu 来降低 RTI 距离，而不一定形成 B-O 和 N-C 键。

证据：`product_core_rmsd = 1.915 A`（核心原子大致在产物位置附近）但 `B-O = 3.144 A` 和 `N-C = 4.007 A`（特定键未形成）。

**2. 自适应 sigma 并未阻止温度升高**

设计预期自适应 sigma（~1/r² 力衰减）能防止温度失控。实际表现：
- 零初速度 → step 1 时 T=0.18K
- 恒温器试图加热到 300K（step 1 时 lambda=1.14）
- Bias 持续注入能量（a 从 0 增长到 -2.0）
- 到 step 1000，T=1163K——高于 RC-MD 的 614K

偏置势达到 -1.21 Ha（~33 eV），远超化学键能。温和的 ~1/r² 衰减在远处有帮助，但振幅的线性增长弥补了衰减，累积的能量注入压倒了恒温器的散热能力。

**3. 1000 步对 RTI 收敛不够**

RTI 距离从 33.5 降到 19.9（下降 40%），说明 bias 确实在拉近碎片。但收敛速度比 RC-MD（从 7.97 到 0.34）慢得多。RTI 距离指标对特定成键坐标天生不够敏感，需要更多步数。

### 对比总结

| 方面 | RC-MD 谐振子 | RTIP 高斯 |
|------|-------------|----------|
| 反应坐标 | 2 个特定距离 | 1 个全局 RMSD |
| 目标知识 | 利用产物键长 | 利用产物完整几何 |
| 1-tBu__CO2 结果 | **成功**（B-O=1.44, N-C=1.66） | 失败（B-O=3.14, N-C=4.01） |
| 温控 | 614K | 1163K（更差） |
| 收敛速度 | 快（rti_dist=0.34） | 慢（rti_dist=19.9） |
| 通用性 | 低（需已知产物） | 高（任意参考几何） |

### 结论

对于产物结构已知、能识别特定成键距离的反应，**RC-MD 谐振子 restraint 严格优于 RTIP Attractive bias。** RTIP 的优势——无需原子对知识的通用性——在有该知识的情况下反而成为劣势。

RTIP 高斯 bias 更适合：
- 产物结构未知的反应
- 无特定键长目标的一般小分子对接
- Repulsive 模式（推离反应物盆地以探索结合态）
- Synthesis 模式（多碎片聚合，无特定成键目标）
