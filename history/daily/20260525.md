## 设计方案

### 核心思路：自动检测替代硬编码

不对每个反应单独指定原子索引和重排映射，而是利用元素类型和距离几何启发式自动检测。

### 小分子类型识别

根据片段 2（`2.xyz`）的元素组成自动判断：

| 原子数 | 元素计数 | 类型 | B 成键元素 |
|--------|----------|------|-----------|
| 3 | C:1 O:2 | CO2 | O |
| 3 | C:1 S:2 | CS2 | S |
| 4 | C:1 O:1 H:2 | H2CO | O |
| 6 | C:2 N:1 H:3 | MeCN | N |
| 11 | C:3 N:1 H:7 | MeCH=NMe | N |

### 反应活性原子自动检测

**在 IS 体系（片段1 + 片段2 拼接）中**：

- **B 原子**：唯一的硼原子，直接通过 `atom_type == Element.B` 找到
- **N2 对**：片段1（1-R-N2 复合物）中距离 B 最近的两个 N 原子。所有体系片段1 都有 4 个 N 原子（2 个 NHC 环 N + 2 个 N2），N2 的两个 N 直接与 B 成键，因此距 B 最近
- **小分子 C**：片段2 中的碳原子。对 CO2/CS2/H2CO 只有一个 C；对 MeCN（2个C）/ MeCH=NMe（3个C），取距离 B 最远的那个 C（亲电碳离反应中心更远）
- **小分子 X（与B成键的原子）**：片段2 中与小分子类型匹配的元素（O/S/N），按距离 B 排序

**在参考结构（product.xyz / ts.xyz）中**：

- **B 原子**：同上
- **N2 对**：用最短 N-N 距离识别（N2 三键 ~1.2 A，远小于其他任何 N-N 距离）。这个方法对含 N 小分子（MeCN, MeCH=NMe）也正确，因为 N2 三键始终是体系中最短的 N-N 距离
- **小分子 C**：所有 C 原子中距离 N2 对最近的那个（在 product/TS 中 N2 端氮与小分子 C 已形成或正在形成 N-C 键）
- **小分子 X**：排除 N2 原子后，距离 B 最近的 O/S/N 原子

### 核心 RMSD 计算

不做全原子重排，而是提取 5-6 个反应核心原子（B + N2×2 + 小分子C + 小分子X×1~2），在模拟帧和参考结构之间分别检测这些原子，然后做 Kabsch 对齐 RMSD。

### RC Restraint 构建

从目标参考结构（product.xyz）中取：
- N-C restraint：N2 端氮与小分子 C 的最近距离
- B-X restraint：B 与小分子 X 原子的最近距离

两个 harmonic distance restraint 使用相同的 k 值。

## 调适过程中发现和修复的问题

### 问题 1：本地无 JAX 环境

`rcmd_lib.py` 最初 `import jax.numpy as jnp` 用于坐标拼接，但本地没有 JAX。修复：改用 `numpy` 做拼接（`np.concatenate`），`System` 类接受 numpy 数组作为 `coord` 参数。JAX 相关导入只在 `run_rcmd.py`（runner）中保留，该文件只在 n5 的 deepmd 环境中运行。

### 问题 2：MeCH=NMe 片段元素签名错误

初始签名写为 `C:3, N:2, H:7`（共12原子），但实际 MeCH=NMe（CH3-CH=N-CH3）为 C3H7N（C:3, N:1, H:7 = 11原子）。修复后正确识别。

### 问题 3：含 N 小分子的参考 N2 检测冲突

MeCN 和 MeCH=NMe 体系中共有 5 个 N 原子（4 个来自 1-tBu 片段 + 1 个来自小分子）。原启发式"距 B 最近的两个 N = N2 对"在 MeCN 体系中失败：小分子 N 在 product 中也与 B 成键，排序时插入到了两个 N2 N 之间。

修复：改用最短 N-N 距离识别 N2 对。N2 三键长度 ~1.2 A，远小于其他任何 N-N 距离（NHC 环 N→N2 > 2.5 A, MeCN N→N2 > 2.5 A）。对所有体系（包括纯 CO2 体系，共 4 个 N）均正确。

同时，小分子 X 检测中显式排除已识别的 N2 原子，避免将 N2 N 误判为小分子反应 N。

### 问题 4：IS 体系中 small_x_indices 排序不确定性

在 IS 几何中，CO2 的两个 O 原子距离 B 近似相等（~5 A），`sorted()` 的稳定性和坐标微小差异导致顺序在 `(34,35)` 和 `(35,34)` 之间不确定。这不影响实际计算，因为 `_min_b_x()` 取所有 small_x_indices 的最小距离，且 RC restraint 使用参考结构中的具体原子对（由 `_closest_pair` 在 reference 中确定）。

## 讨论与决策记录

1. **不做全原子重排映射**：原 runner 的 `REFERENCE_REORDER_1TBU_CO2` 是手动推导的 36 原子排列。对每个新反应重复此过程成本高且易错。改用"在参考结构和模拟结构中独立检测反应核心原子"的方案，只需 5-6 个原子的元素匹配，不需要知道完整排列。

2. **保留原有文件不改动**：`run_bias_exploration_1tbu_co2.py` 和 `analyze_bias_exploration_1tbu_co2.py` 作为已验证的参考锚点保留。如果泛化版本出现问题，仍可用原版本独立验证 1-tBu__CO2。

3. **k=0.006 对所有反应统一使用**：该力常数单位为 Hartree/Bohr^2，仅与距离约束相关，与体系总原子数无关。理论上应可跨体系迁移。如果某些体系结果不理想，再考虑单独调参。

4. **不使用 TS target**：基于昨天的结论，product target 的 B-X 和 N-C 距离方向一致，RC restraint 可以同时有效约束。TS target 的两个距离方向不一致，轨迹不稳定。
